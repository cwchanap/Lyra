# Chapter 1 Actual Writing Agent Addendum
# 第 1 章《雨鐘咖啡館殺人事件》實際寫作補充規範

> 適用對象：負責把 `chapter1_final_writing_plan_revised_v3_7.md` 轉成 detailed outline、scene draft、調查互動稿、審查會辯論稿、證物文字、UI 文案與角色台詞的 writer agent。  
> 本文件不是新 canon，不改案件邏輯。它只補「如何把 V3.7 寫成可玩的章節，而不是說明書」。

---

## 0. 寫作總目標

第 1 章的成稿要讓玩家得到三種感覺：

1. **我救下了一個普通人。**  
   三宅不是抽象嫌疑人，而是一個會怕被開除、會買錯飲料、會把蛋糕邊帶回家的年輕人。

2. **我親手拆掉了一個錯誤故事。**  
   玩家不是聽相馬解釋真相，而是透過現場走位、證物包、審查會選擇與 replay，自己理解 summary 哪裡錯。

3. **案子結了，但雨裡還有東西。**  
   `ZW_A16.lock`、藍色透明傘、金木犀、雨宮訊息只留下形狀，不要解釋來源。

本章最重要的句子不是角色台詞，而是玩家心中的理解：

> 「門鎖沒有說謊。是人把它說成了另一個故事。」

## 1. Writer Agent 不可改動的邊界

### 1.1 不可改案件骨架

以下內容一律不得改：

- 死者：增田圭。
- 表面嫌疑人：三宅蒼太。
- 真兇：北見修一。
- 真兇動機：北見盜賣 KAGAMI 試點資料，增田準備檢舉。
- 詭計核心：門鎖在維護模式只保留本機順序；summary 把伺服器合併時間當成事件時間。
- 結論：raw / 本機順序沒有造假；錯的是 summary 的排列與解讀。

### 1.2 不可提前揭露主線

第 1 章禁止：

- 說出 A-90、A90 Tag、A-90 Hold。
- 說出 ZERO WITNESS 全名。
- 把 `ZW_A16.lock` 拆字。
- 解釋青葉舊案。
- 讓雨宮現身。
- 讓藍傘被鑑定。
- 讓金木犀成為證物。
- 讓相馬出現明確青葉閃回。

### 1.3 不可把技術寫成講義

角色台詞只用三個玩家可懂詞：

| 玩家可聽見詞 | 內部含義 |
|---|---|
| 摘要 | KAGAMI summary / report layer |
| 本機順序 | maintenance panel / local event order |
| 核准片段 | limited raw extraction / narrow extraction |

避免在台詞中反覆使用：raw、sync layer、foreground timestamp、Narrative Anchor Rule、queued item、metadata integrity 等術語。

玩家只需要聽懂一句：

> 「摘要抓到的，不一定是第一個人；它抓的是最好對得上的那個。」

## 2. 成稿交付格式建議

Writer agent 的輸出建議分成六份，避免一份大稿混雜所有功能。

### 2.1 Scene List

每場 5～8 行即可：

```markdown
### Scene 07：第一次現場調查 — 後場白板
- 位置：雨鐘後場入口。
- 玩家目標：理解閉店流程是 routine，不是案發夜巧合。
- 玩家可見：白板上「維護紀錄冊確認」「關閉維護模式」未勾選。
- 新增資料：人物筆記 / 場景記憶，不立即作正式證物。
- 禁止：不要讓相馬當場說出 23:20 發現理由。
```

### 2.2 Scene Draft

每場正式 scene draft 建議包含：

```markdown
## Scene 名稱

### 功能
這場在故事、推理、角色、節奏中的作用。

### 進場狀態
玩家已知什麼；角色以為什麼是真的。

### 場景描寫
只寫玩家能看見、聽見、操作到的內容。

### 對話草稿
台詞以角色聲音為主，不要說明書化。

### 調查互動 / 選項
可調查點、玩家點錯時得到的生活資訊、點對時得到的推理資訊。

### 證物 / 人物筆記更新
只列真正新增或更新的項目。

### 出場狀態
玩家此時應該多知道什麼、誤會什麼、仍不知道什麼。
```

### 2.3 Investigation Interaction Script

調查點不要每個都給硬線索。建議比例：

| 類型 | 比例 | 功能 |
|---|---:|---|
| 破案資訊 | 40% | 直接支撐三個證據包 |
| 角色生活資訊 | 30% | 讓三宅、店長、增田、早坂、神谷像活人 |
| 氣氛與伏筆 | 20% | 藍傘、金木犀、雨聲、店內物件 |
| 錯誤焦點 / 紅鯡魚 | 10% | 讓玩家短暫懷疑店長、片瀨或三宅更深謊言 |

### 2.4 Hearing / Review Debate Script

審查會每一 phase 只讓玩家回答一個問題。

不要在同一輪同時問：死亡時間、門鎖時間、北見身份、雨宮訊息、外包鏈。

### 2.5 Evidence Text

每個證物要有三層文字：

1. **初始描述**：玩家剛取得時看到的中性說法。
2. **更新後描述**：玩家發現新意義後追加一句。
3. **審查會使用語**：出示時相馬能說的 1～2 句。

### 2.6 Scene QA Checklist

每場寫完後，writer agent 必須自檢：

- 這場是否只服務 1～2 個主要功能？
- 是否有角色在替主題演講？
- 玩家是否能看見線索，而不是只聽解釋？
- 這場是否新增了不必要的新問題？
- 是否誤把伏筆寫成證物？

## 3. 三個證據包的寫作方式

第 1 章不要讓玩家記二十多個證物。玩家前台只要記三個證據包。

---

### 3.1 三宅小謊包

核心命題：

> 三宅有說謊，但不是因為殺人。

包含：

- 偷偷打電話給母親。
- 去後場拿蛋糕盒。
- 閉店 routine 中本來就會進後場。
- 熱牛奶、飯糰袋、母親胃不好。

寫作重點：

- 三宅不是無辜聖人。他確實隱瞞。
- 他的謊要讓玩家一開始也不太信他。
- 但謊的理由要很生活：怕店長以為他偷東西、怕被開除、怕母親擔心。

錯誤寫法：

> 「我真的不是兇手！我只是很孝順！」

較好寫法：

> 三宅看著自動販賣機吐出的黑咖啡，愣了幾秒。  
> 「我本來要買熱牛奶的。」  
> 相馬：「給你自己？」  
> 三宅搖頭。  
> 「我媽最近胃不太好。可是我每次一緊張，就會買錯。」

---

### 3.2 第三者動線包

核心命題：

> 有人比三宅更早進入後場，而且走的是承包商動線。

包含：

- L 型後場視線遮蔽。
- 高貨架、半掩防火門、未亮感應燈。
- 地板雨水乾燥圖。
- 濕傘套。
- External Maintenance Credential。
- 臨時維護工單。
- `K.` 與外包人員弱曝光。

寫作重點：

- 雨水與傘套不要直接定罪北見。
- 它們只證明「第三者比三宅更早走過這裡」。
- 北見身份要靠工單、外包憑證、增田備忘、帳號審核壓力一起收束。

錯誤寫法：

> 「這個傘套是北見的，所以北見是兇手。」

較好寫法：

> 相馬：「這只說明一件事。」  
> 早坂：「有人比三宅更早進來？」  
> 相馬點頭。  
> 「而且不是從員工動線。」

---

### 3.3 門鎖時序包

核心命題：

> 門鎖沒有說謊，但 summary 把合併時間讀成事件時間。

包含：

- 閉店 SOP 白板。
- 店長 maintenance panel 截圖，作 lead。
- 黑瀨 / 鑑識固定本機順序。
- 審查會核准片段。
- 89.7 秒合併延遲。
- staff credential + CCTV + 打卡紀錄讓三宅成為最好看的時間錨。

寫作重點：

- 不要說 KAGAMI 造假。
- 不要說門鎖錯了。
- 要讓玩家理解「正確材料被排成錯誤故事」。

審查會核心台詞可用：

> 神谷：「門鎖紀錄沒有被改。」  
> 相馬：「對。」  
> 神谷：「那你在否定什麼？」  
> 相馬：「否定我們替它補上的意思。」

## 4. 角色聲音指南

### 4.1 相馬律

狀態：年輕、認真、有證據信仰，但不是天才神探。

台詞特徵：

- 喜歡把事情整理成順序。
- 前半會相信報表完整。
- 後半開始從現場反推。
- 不說漂亮大話，結論偏短。

可用語感：

> 「先不要急著判斷。」  
> 「這不是矛盾，是我們還沒看懂它。」  
> 「門鎖說了事實。但我們聽成了故事。」

避免：

- 過度冷酷。
- 一眼看穿所有事。
- 一章內變成反 KAGAMI 鬥士。

### 4.2 早坂茜

狀態：程序感強、語氣直接、不是導師型角色。

台詞特徵：

- 會用實務話壓住相馬。
- 重視保全鏈、紙本、審查資格。
- 不要讓她老是講主題句。

可用語感：

> 「這不是預演。今天輸了，資料入口就關了。」  
> 「截圖只能讓我們懷疑，不能讓我們贏。」  
> 「你要救他，就拿可以被採用的東西救。」

避免：

- 變成劇情說明員。
- 過早揭露青葉或私人動機。

### 4.3 三宅蒼太

狀態：可疑、緊張、生活壓力大，但不是笨蛋。

台詞特徵：

- 話容易斷。
- 會先保護母親和工作，而不是保護自己。
- 說謊時不是表演型，而是小聲省略。

可用語感：

> 「那個……我不是不想說。」  
> 「如果店長知道，我可能就不能繼續做了。」  
> 「我媽一直以為我在店裡很可靠。」

避免：

- 過度可憐。
- 一開始就像百分百無辜。
- 把他的孝順寫成煽情標籤。

### 4.4 神谷澪

狀態：制度守門人，不是反派。

台詞特徵：

- 理性、精準、壓迫感強。
- 相信 summary 是因為她知道人類偏見很可怕。
- 最後可以承認讀法有問題，但不要自我否定。

可用語感：

> 「嫌疑人的態度不是證據。」  
> 「我不接受『他看起來不像兇手』這種反駁。」  
> 「你現在推翻的不是紀錄，是紀錄被放進故事裡的方式。」

避免：

- 變成嘲諷型檢察官。
- 太早站到相馬方。
- 說出完整角色成長方向。

### 4.5 北見修一

狀態：膽怯、利己、被壓迫但仍負責任的殺人者。

台詞特徵：

- 不要長篇社會評論。
- 怨言只留一兩句。
- 被逼到證物時才破防。

可用語感：

> 「規則從來不是寫給我們這種人看的。」  
> 「我只是想在被換掉之前，留一點退路。」  
> 「他不該把那份草稿留下。」

避免：

- 把他寫成大反派。
- 把他寫得太可憐，削弱殺人責任。
- 讓他說出 A-90、青葉、第零證人。

### 4.6 黑瀨徹

狀態：現場感代表，不懂太多科技但懂人和地面。

台詞特徵：

- 說話偏粗，但不笨。
- 看鞋底、水痕、傘架、門口。
- 不替玩家破案，只給現場觀察。

可用語感：

> 「人進來，總會把外面的東西帶進來。」  
> 「這水痕不像剛剛才踩的。」  
> 「截圖我不收。要用，就讓鑑識固定。」

## 5. 節奏與場景長度規範

### 5.1 每段建議長度

| 段落 | 成稿目標 |
|---|---:|
| Beat 0 冷開場 + 人性鉤子 | 3～5 分鐘 |
| Beat 1～2 事務所與委託 | 15～20 分鐘 |
| Beat 3 初次現場 | 30～35 分鐘 |
| Beat 4 三宅詢問 | 15～20 分鐘 |
| Beat 5 第一輪審查會 | 25～30 分鐘 |
| Beat 6 喘息 | 8～12 分鐘 |
| Beat 7 反轉調查 + replay | 30～40 分鐘 |
| Beat 8 北見調查 | 20～25 分鐘 |
| Beat 8.5 整理點 | 5～8 分鐘 |
| Beat 9 最終審查會 | 35～45 分鐘 |
| Beat 10 生活收束 + 鉤子 | 12～18 分鐘 |

### 5.2 第一幕轉折不要太晚

第一個明確壓力點要在玩家進入 30～40 分鐘內出現：

> 今天審查會過不了，三宅會正式起訴，資料入口會關閉。

不要讓玩家玩了很久才知道「現在為什麼急」。

### 5.3 日常場景要短，但有物件記憶

每個日常段落都要有一個可記憶物件：

| 日常場景 | 物件 |
|---|---|
| 相馬事務所 | 壞掉的咖啡機、溫度計、紙本收據 |
| 三宅母親 | 被捏皺的飯糰袋 |
| 三宅詢問 | 買錯的黑咖啡 / 原本想買熱牛奶 |
| 店長 | 舊掛鐘、閉店白板 |
| 章末 | 太甜的金木犀拿鐵、未取走的藍色透明傘 |

日常場景的目的不是拖時間，而是讓玩家記住：summary 壓縮掉的是人。

## 6. Scene Writing Do / Don't

### 6.1 Do

- 用玩家走位證明三宅看不到屍體。
- 用未勾白板提前種下 23:20 發現理由。
- 用黑瀨固定 maintenance panel，保護程序可信度。
- 用 Beat 8.5 讓玩家先停在「第三者」而不是直接跳到「北見」。
- 讓神谷每次反駁都合理。
- 讓三宅小謊真的讓玩家不舒服。
- 讓北見被指出時不是突然出現，而是前面細節逐步合上。
- 讓章末先完成救人的情緒，再丟 `ZW_A16.lock`。

### 6.2 Don't

- 不要讓角色解釋整套 KAGAMI 三層。
- 不要把雨宮訊息寫成提示 UI。
- 不要讓藍傘被任何人拿起來研究。
- 不要讓金木犀觸發閃回。
- 不要讓咖啡機成為死亡時間硬定時器。
- 不要讓店長像工程師。
- 不要讓北見像 mastermind。
- 不要讓審查會只剩相馬長篇演說。
- 不要把每個伏筆都拍成「請記住我」。

## 7. 審查會寫作規範

### 7.1 每輪攻防格式

每輪審查會建議固定使用：

```markdown
1. 對方主張
2. 玩家需要指出的問題
3. 可出示證物
4. 錯誤選擇時的合理反駁
5. 正確選擇後的新問題
```

這樣審查會會像遊戲，而不是相馬朗讀推理。

### 7.2 錯誤選項要合理

錯誤選項不能只是「明顯錯」。例如：

- 玩家太早出示北見資料 → 神谷反駁：「你還沒推翻三宅的時間錨。」
- 玩家只靠舊掛鐘 → 神谷反駁：「掛鐘可以早就壞了。」
- 玩家說 KAGAMI 造假 → 神谷反駁：「你沒有證據證明 raw 被改。」
- 玩家用三宅人品辯護 → 神谷反駁：「人品不是不在場證明。」

錯誤反駁要教玩家正確思路。

### 7.3 最終審查會不應一口氣爆全部

最終審查會要分三步收束：

1. 三宅時間錨失效。
2. 第三者更早進入成立。
3. 第三者身份收束到北見。

不要一開始就說「北見是真兇」。

## 8. 伏筆留白規範

### 8.1 90 秒

可見：`89.7s`、維護合併延遲、大概九十秒。  
不可說：A-90、舊常數、青葉來源。

### 8.2 雨宮

可見：匿名訊息、低解析裁切圖 / 外包縮圖一角。  
不可說：她是誰、她怎麼進系統、她是否在幫相馬。

### 8.3 藍傘

可見：傘架裡無人認領的藍色透明傘。  
不可做：鑑定、姓名貼、兒童病房暗示、正式證物。

### 8.4 金木犀

可見：菜單、增田常點、章末太甜拿鐵、相馬手停半拍。  
不可做：頭痛、閃回、創傷說明、推理點。

### 8.5 `ZW_A16.lock`

可見：檔名、無法開啟、不是北見盜賣案所需資料。  
不可做：拆字、解碼、青葉對接、第零證人。

---

## 9. Actual Writing Pass 順序

建議 writer agent 不要直接寫完整章。先分五 pass。

### Pass 1：Scene Skeleton

只列場景順序、場景功能、進出狀態。不要寫長台詞。

### Pass 2：Evidence Flow

確認三個證據包在 scene skeleton 中都有足夠可見 seed：

- 三宅小謊包。
- 第三者動線包。
- 門鎖時序包。

### Pass 3：Dialogue Draft

開始寫台詞，但每場限制：

- 主題句最多 1 句。
- 技術解釋最多 2 句。
- 每個角色發言要符合聲音指南。

### Pass 4：Gameplay Draft

補調查點、錯誤選項、審查會 proof order、證物更新文字。

### Pass 5：Pacing / Silence Polish

刪掉多餘解釋，保留空鏡、停頓、手部動作、物件記憶。

## 10. 最後 QA Checklist

成稿交付前逐條檢查：

### 案件公平性

- 玩家能否在最終審查會前看見所有必要線索？
- 北見是否早有弱曝光，而不是終盤新人物？
- 三宅看不到屍體是否透過走位 / replay 證明？
